関数: `DataInterface.get_horse_master()`

- mykeibadb: `KYOSOBA_MASTER2`
- scraping:

## 差分A: プロバイダー間の既知差分

| カラム | 内容 | 血統登録番号の例 |
|--------|------|------|
| 東西所属コード | 所属が変わった場合、scrapingは現在の所属、mykeibadbは当時の情報 | "2016103690" |
| 調教師コード | 転厩した場合、scrapingは現在の調教師、mykeibadbは当時の情報 | "2016103690" |
| 調教師名略称 | 表記ゆれあり | "2016103690" |
| 生産者名 | 表記ゆれあり | "2018103559" |
| 馬主名 | 表記ゆれあり | "2021103695" |
| 父父馬名・父母馬名 | 外国馬はmykeibadbに日本語馬名が登録されていない | "2021190001" |
| 着回数 | mykeibadbは地方競馬のデータが不足する場合あり | "2021103695" |

## 差分B: mykeibadbテーブルとの差分

| カラム | 内容 |
|--------|------|
| 調教師名略称/馬主名 | 全角スペース（U+3000）を除去 |

## 差分B: カラム名について（3代血統情報）

mykeibadb（KYOSOBA_MASTER2）の<3代血統情報>のカラム名はKETTO1〜KETTO14の番号で管理されており、それぞれ「繁殖登録番号」「馬名」が対応している。統一スキーマでは以下のように日本語カラム名に変換している。

| mykeibadbカラム名 | 統一スキーマのカラム名 |
|------|------|
| KETTO1_HANSHOKU_TOROKU_BANGO | 父繁殖登録番号 |
| KETTO1_BAMEI | 父馬名 |
| KETTO2_HANSHOKU_TOROKU_BANGO | 母繁殖登録番号 |
| KETTO2_BAMEI | 母馬名 |
| KETTO3〜KETTO14（省略） | 父父〜母母母（繁殖登録番号と馬名） |
